# 堆叠分支工作流

在处理大型分支时，将其分解为较小的部分通常很有用，为每个独立的更改块创建单独的分支会很有帮助。例如，你可以为准备性重构创建一个分支，为后端更改创建一个分支，为前端更改创建另一个分支。这些分支将依次堆叠在一起。

Git 支持将这样的堆叠分支作为一个整体进行 rebase；你可以通过将 git 配置 `rebase.updateRefs` 设置为 true 来启用它。如果你随后对堆叠的最顶层分支进行 rebase，堆叠中的其他分支将跟随。这包括交互式 rebase，因此例如修改堆叠中第一个分支的提交将"正常工作"，因为它会保持其他分支正确地堆叠在其上。

Lazygit 通过用青色星号（如果你使用 [nerd fonts](Config.md#display-nerd-fonts-icons) 则为青色分支符号）标记堆叠中的各个分支头来可视化它们。
