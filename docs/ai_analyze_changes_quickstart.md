# AI 智能变更分析 - 快速开始

## 使用场景

当你有大量文件变更时，一次性发送所有 diff 给 AI 可能会超出上下文限制。`analyze_changes` 工具会自动：

1. 逐个文件分析
2. 整合结果
3. 生成结构化报告

## 快速示例

### 示例 1：分析工作区变更

在 AI Chat 中输入：

```
帮我分析当前的变更
```

AI 会自动调用 `analyze_changes` 工具，输出类似：

```
# 变更分析报告

**文件数**: 3 个（成功分析 3，失败 0）
**总变更行数**: 约 150 行

## 详细分析

### 1. pkg/gui/ai_chat.go
添加了 AI 聊天输入处理逻辑，实现了基本的键盘交互。
建议添加输入长度限制。

### 2. pkg/gui/controllers/helpers/ai_chat_helper.go
实现了聊天会话管理和消息渲染功能。
代码结构清晰，未发现明显问题。

### 3. pkg/gui/controllers/helpers/ai_chat_helper_test.go
添加了完整的单元测试，覆盖了核心功能。
```

### 示例 2：分析暂存区变更（关注安全）

```
分析暂存区的变更，重点关注安全问题
```

输出会聚焦于安全相关的分析。

### 示例 3：分析特定方面

```
分析当前变更，重点关注性能优化
```

或

```
分析暂存区，重点关注代码质量
```

## 工作原理

```
┌─────────────────────────────────────────────────┐
│  用户: "分析当前的变更"                          │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  AI Agent 调用 analyze_changes 工具             │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  获取变更文件列表                                │
│  - file1.go (50 行变更)                         │
│  - file2.go (30 行变更)                         │
│  - file3.go (70 行变更)                         │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  逐个文件分析 (避免上下文溢出)                   │
│                                                 │
│  file1.go → AI 分析 → 结果1                     │
│  file2.go → AI 分析 → 结果2                     │
│  file3.go → AI 分析 → 结果3                     │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  整合所有分析结果                                │
│  - 统计信息                                     │
│  - 逐文件详情                                   │
│  - 整体建议                                     │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  返回结构化报告给用户                            │
└─────────────────────────────────────────────────┘
```

## 优势

### 1. 突破上下文限制

传统方式：
```
所有文件 diff (5000 行) → AI (上下文溢出 ❌)
```

智能分析：
```
file1 (50 行) → AI → 结果1 ✓
file2 (30 行) → AI → 结果2 ✓
file3 (70 行) → AI → 结果3 ✓
整合 → 完整报告 ✓
```

### 2. 更精准的分析

每个文件独立分析，AI 可以更专注于单个文件的细节。

### 3. 灵活的分析重点

可以根据需求调整分析重点：
- 安全问题
- 性能优化
- 代码质量
- 架构设计
- 等等...

## 配置选项

### 参数说明

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `staged` | bool | 是否分析暂存区 | false |
| `focus` | string | 分析重点 | "" (全面分析) |

### 使用示例

```go
// 在代码中直接调用
tool := NewAnalyzeChangesTool(deps, provider)
result := tool.Execute(ctx, tools.ToolCall{
    ID: "analyze-1",
    Name: "analyze_changes",
    Params: map[string]any{
        "staged": true,
        "focus": "安全问题",
    },
})
```

## 常见问题

### Q: 分析需要多长时间？

A: 取决于文件数量和 AI 响应速度。通常每个文件 1-3 秒，10 个文件大约 10-30 秒。

### Q: 可以取消正在进行的分析吗？

A: 可以。分析过程支持 context 取消，可以随时中断。

### Q: 单个文件分析失败会怎样？

A: 不会影响其他文件的分析。失败的文件会在报告中标注错误信息。

### Q: 支持哪些文件类型？

A: 所有文本文件都支持。AI 会根据文件扩展名和内容自动识别语言。
