# lazygit 二次开发计划

## 目标

在 lazygit 基础上新增两大功能模块：
1. **命令行输入区**：在 TUI 内直接输入并执行 git 命令
2. **AI 集成**：以 AI 生成 commit message 为切入点，逐步扩展

---

## 模块一：命令行输入区

### 功能描述

在界面底部新增一个可输入的命令栏，用户按下特定快捷键后光标跳入，
输入任意 git 命令（或 shell 命令）后回车执行，结果输出到 CommandLog 面板，
执行完毕自动刷新界面。

**交互设计：**
```
[Status] [Files] [Branches] [Commits] [Stash]
┌─────────────────────────────────────────────┐
│  (各面板内容)                                │
└─────────────────────────────────────────────┘
┌─ Command ───────────────────────────────────┐
│ > git rebase -i HEAD~3_                     │  ← 新增区域
└─────────────────────────────────────────────┘
┌─ Command Log ───────────────────────────────┐
│  git status                                 │
└─────────────────────────────────────────────┘
```

**快捷键：**
- `:` 或 `>` —— 全局呼出命令输入框（参考 vim 命令模式）
- `Enter` —— 执行命令
- `Esc` —— 关闭输入框，返回上一个面板
- `↑/↓` —— 浏览命令历史

### Phase 2：Commit Message 生成

#### 功能描述

在 commit message 输入界面，按下 `<AI触发键>`（默认 `Ctrl+g`），
lazygit 自动获取当前 staged diff，调用 AI 生成 commit message，
结果预填充到输入框，用户可继续编辑后提交。

**交互流程：**
```
用户 staged 文件
→ 按 c 进入 commit message 输入框
→ 按 Ctrl+g 触发 AI
→ 底部状态栏显示 "AI 正在生成..."
→ 生成完成，commit message 自动填入输入框
→ 用户确认/修改后 Enter 提交
```

#### Prompt 设计

```
你是一个 Git 提交信息生成助手。
请根据以下 staged diff，生成一条符合 Conventional Commits 规范的提交信息。

规范：<type>(<scope>): <subject>

要求：
- type 从 feat/fix/docs/style/refactor/perf/test/chore 中选择
- subject 用中文，简洁描述变更内容，不超过 50 字
- 只输出提交信息本身，不要解释

diff:
{{DIFF}}
```

#### 涉及文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `pkg/gui/controllers/helpers/commits_helper.go` | 修改 | 新增 `AIGenerateCommitMessage()` 方法 |
| `pkg/gui/controllers/commit_message_controller.go` | 修改 | 注册 `Ctrl+g` 快捷键，调用 helper |
| `pkg/gui/types/common.go` | 修改 | CommonHelper 中注入 AI Client |
| `pkg/i18n/i18n.go` | 修改 | 添加 `AIGenerating`、`AIError` 等提示文字 |

#### 核心实现

```go
// pkg/gui/controllers/helpers/commits_helper.go

func (self *CommitsHelper) AIGenerateCommitMessage() error {
    // 1. 检查 AI 是否启用
    if !self.c.UserConfig().AI.Enabled {
        return self.c.ErrorMsg("AI 未启用，请在配置文件中设置 ai.enabled: true")
    }

    // 2. 显示加载状态
    self.c.SetStatus("AI 正在生成 commit message...")

    // 3. 获取 staged diff
    diff, err := self.c.Git().Diff.GetStagedDiff()
    if err != nil {
        return err
    }
    if diff == "" {
        return self.c.ErrorMsg("没有 staged 的变更")
    }

    // 4. 异步调用 AI（不阻塞 UI）
    go func() {
        message, err := self.c.AI().GenerateCommitMessage(diff)
        self.c.OnUIThread(func() error {
            if err != nil {
                return self.c.ErrorMsg("AI 生成失败: " + err.Error())
            }
            // 5. 填入 commit message 输入框
            self.c.Helpers().Commits.UpdateCommitPanelView(message)
            self.c.SetStatus("")
            return nil
        })
    }()

    return nil
}
```

### Phase 3：自然语言转 git 命令

基于命令输入区，识别非 git 开头的输入作为自然语言，
调用 AI 转换为实际命令，填入输入框（不直接执行，需用户确认）。

**示例：**
```
> 把上个提交合并进来    →  AI 转换 →  git rebase -i HEAD~2
> 撤销最近3次提交       →  AI 转换 →  git reset HEAD~3
```

**Prompt 设计：**
```
将以下自然语言描述转换为 git 命令。
只输出命令本身，不要解释，不要 markdown 格式。

描述: {{INPUT}}
```

## 依赖

- 外部依赖：仅需标准库 `net/http`，无需引入新的第三方包
- AI 服务：OpenAI API 或本地 Ollama（推荐后者，免费且隐私安全）

## 风险与注意事项

1. **API Key 安全**：配置文件中的 key 支持 `${ENV_VAR}` 引用，不硬编码
2. **网络超时**：AI 调用必须异步，不能阻塞 UI 主线程
3. **上游同步**：建议维护补丁集方式，定期 rebase 到上游新版本
4. **diff 过大**：staged diff 超过 token 限制时，自动截断并提示用户
